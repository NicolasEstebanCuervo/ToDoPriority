# Bienvenido

Storybook te ayuda a construir componentes de UI de forma aislada de la lógica de tu aplicación, datos y contexto. Esto facilita el desarrollo de estados difíciles de alcanzar. Guarda estos estados de UI como **stories** para revisar durante el desarrollo, pruebas o control de calidad.

Explora ejemplos de stories navegando por ellos en la barra lateral. Puedes ver el código en el directorio `stories` para aprender cómo funcionan. Recomendamos construir interfaces de usuario utilizando un enfoque [**component-driven**](https://componentdriven.org), comenzando con componentes atómicos y terminando en páginas.
